# [**if/else и тернарный оператор**](https://metanit.com/sharp/tutorial/2.5.php)

**Конструкция** и **тернарный** оператор нужны **для ветвления логики** и выбора варианта поведения программы или **присвоения значения** переменной **в зависимости от** каких-то **условий**.

## **if/ else if/ else**  
Основной элемент: `**if**(условие){что делаеть если условие **true**}`

Представляет собой **bool проверку**, выполнено ли **условие**. **Если выполнено**, **запускается** блок кода. **Если не выполнено**, блок **игнорируется**. Можно использовать отдельно от остальных, проверяя одно конкретное условие.

Промежуточный элемент: `**else if**(условие){блок кода}`

Такая же проверка, как **if**. Используется, если вариантов более 2\. Можно вообще не использовать, если не нужно.

Закрывающий элемент; `**else**{блок кода}`

Такой же **необязательный** элемент, как else if. Но **не имеет условия** срабатывания. Он автоматически срабатывает, если не срабатывает if и промежуточные else if. Нужен либо когда вариантов всего 2, либо для обработки ошибок и исключений, чтобы программа не упала в случае, если предыдущие условия критически важны, но почему-то не выполнены.Также используется просто как финальный элемент конструкции, чтобы не тратить ресурсы на проверку условия.

**Тернарная операция**  
`условие ? вариант 1(условие выполнено) : вариант 2 (условие не выполнено)`

Также выбирает из вариантов в вариантов в зависимости от условия. Но есть несколько принципиальных различий с конструкцией if.

**Возвращение значения**  
Терновый оператор, это выражение. Не выполняет блок кода, а возвращает значения. Поэтому у него должно быть куда это значение вернуть. Сделать можно в:

**Переменную**: `var x = тернарный оператор`  
**Метод Write**: `Console.Write(терный оператор выбирает строку) ` 
**Возврат значения return**: `return тернарный оператор`  
**Часть выражения**: `x = 5 + (тернарный оператор)`  
**Аргумент**: `bool x = (тернарный оператор) && y != 0			//считает плохим кодом`  
**Инициализация** полей, свойств, аргументов конструктора внутри метода

**Виды значений**  
```
var x \= условие ? число : “строка” 		// ошибка  
var x \= условие ? число : число		// работает
```
**Даже** если переменная **var**, варианты должны быть **одного типа**.

**Вложенность**  
```
var x =  
условие 1 ? вариант 1 :  
условие 2 ? вариант 2:  
вариант 3
```
По сути работает так же, как и **if/ else if/ else** с учетом особенность тернарного оператора. Позволяет усложнить выражение несколькими условиями. Но считается не очень хорошей практикой и плохо читаемым кодом. В таких случаях лучше использовать if.

**Скобки**  
```
var x = y > 4 + 1 ? вариант 1: вариант 2		//выполняет условие как (y > 4) + 1  
var x = y > (4 + 1) ? вариант 1: вариант 2		//выполняет условие как  y > (4 + 1); итого y > 5
```
В условии можно использовать скобки как и в любом математическом выражении. Они имеют более высокий приоритет, чем логические и сравнительные операторы, что меняет результат условия.

**Ограничения**  
Тернарный оператор, в отличии от if, **только возвращает** куда-то значение. Он **не может** быть полноценной **логической веткой** для программы. Тернарный оператор **можно** использовать **внутри** метода Write(), **но** использовать **метод** Console.Write() как **один из вариантов** после проверки условия **нельзя**. Это же касается и других методов о команд.
