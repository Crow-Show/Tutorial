# **[Структура программы](https://metanit.com/sharp/tutorial/1.5.php)**

**С\#** \- язык, ориентированный на **ООП**. Файлы языка имеют разрешение **.cs**. Внутри любой программы по умолчанию запущена библиотека классов **using System.** Она позволяет использовать базовый набор инструкций, таких как **Consol.** А также класс **class Program,** который и представляет собой программу.

Любая программа на **C\#** основана на **ООП** по своей сути, даже если не применяет классы и объекты непосредственно, потому что для работы программы базово требуются классы. По сути своей любой код на **C\#** \- это программа, позволяющая из своих инструментов собрать другую программу внутри блока кода **static void Main()** в фигурных скобках **{}**.

Структура кода в C\# имеет 2 ключевых аспекта:

1. **Строгий порядок выполнения.** Код идет строго **сверху вниз.** Движение сверху вниз для C\# \- временная шкала. Переменная становится доступна для использования только после того, как была объявлена. А именно \- ниже. Выше ее еще не существует, поэтому использовать её нельзя.

2. **Блоки кода**. Фигурные скобки отделяют часть кода от общей временной линии. Блок кода может использовать все, что было выше него, то есть было раньше по хорологии. Но сам не пропускает свои данные ниже. Переменную, объявленную внутри блока, нельзя использовать вне его. Там она не существует.   
   **Код внутри {} по своей сути \- ветка дерева**. Отросток на определенной части ствола. Он может использовать все, что было до него. Но не то, что после. А также ствол выше ветки не может использовать те данные, что на ветке.


Также стоит упомянуть **регистрозависимость.** Инструкции и переменные должны быть названы строго так, как называются, без изменения регистра. Из этого вытекают **правила нейминга C\#**. У каждого типа данные, которые можно объявить, есть свои общепринятые правила нейминга. Помимо общих правил передачи смысла переменной ее названием также есть правила использования регистра:

```CamelCase -``` первая буква всегда маленькая. Если переменная из 2 и более слов \- каждое последующее выделяется большой буквой.  
**Применение**: переменные и методы.  
**Примеры:** ```name \- userName```

```PascalCase``` \- первая буква большая. Последующие слова также выделяются большой буквой.  
**Применение:** классы, пространства имен, свойства, константы.  
**Примеры:** ```Name_1 \- UserName_1_2```

```UPPER_CASE ```- все в верхнем регистре. Разделение слов через нижнее подчеркивание.  
**Применение:** константы(изредка для глобальных)  
**Примеры:** ```NAME - USER_NAME```
